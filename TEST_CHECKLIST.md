# 한눈 가계부 Android 테스트 체크리스트

이 문서는 현재 구현된 Android 앱 기능을 실제 기기, 에뮬레이터, Debug 빌드에서 확인하기 위한 체크리스트입니다.

## 테스트 전 준비

- [ ] Android Studio에서 Gradle Sync가 성공한다.
- [ ] Debug APK가 빌드된다.
- [ ] 실제 기기 또는 에뮬레이터에서 앱이 크래시 없이 실행된다.
- [ ] 앱 이름이 `한눈 가계부`로 표시된다.
- [ ] 앱을 처음 실행했을 때 홈 화면이 표시된다.
- [ ] 하단 `지출 / 홈 / 수입` 고정바가 화면 하단에 붙어 있다.
- [ ] 상단 광고 배너가 무료 상태에서 표시된다.
- [ ] 프리미엄 상태에서는 광고 배너가 숨겨진다.

## 기본 수입/지출 작성

- [ ] 홈 하단 `수입` 버튼으로 수입 작성 화면에 진입한다.
- [ ] 홈 하단 `지출` 버튼으로 지출 작성 화면에 진입한다.
- [ ] 새 작성 화면의 기본 날짜가 현재 기준 날짜로 표시된다.
- [ ] 금액 입력 중 콤마가 자동 표시된다.
- [ ] 금액은 저장 후 숫자 값으로 정상 반영된다.
- [ ] 금액 0원, 빈 제목, 잘못된 날짜 형식에서 에러 메시지가 표시된다.
- [ ] 날짜 직접 입력 `YYYY-MM-DD`가 가능하다.
- [ ] 날짜 선택 버튼으로 DatePicker가 열린다.
- [ ] DatePicker에서 선택한 날짜가 `YYYY-MM-DD`로 입력된다.
- [ ] 수입 작성 화면에는 영수증 사진 UI가 표시되지 않는다.
- [ ] 지출 작성 화면에는 영수증 사진 UI가 표시된다.
- [ ] 저장 후 홈, 상세, 전체보기 화면에 거래가 반영된다.

## 실제 반영 거래

- [ ] `실제 반영` 상태로 과거 날짜 거래를 저장할 수 있다.
- [ ] `실제 반영` 상태로 오늘 날짜 거래를 저장할 수 있다.
- [ ] `실제 반영` 상태로 미래 날짜 거래를 저장할 수 있다.
- [ ] 실제 반영 거래는 `posted` 상태로 저장된다.
- [ ] 실제 반영 수입은 홈 월 정산의 수입 합계에 포함된다.
- [ ] 실제 반영 지출은 홈 월 정산의 지출 합계에 포함된다.
- [ ] 계좌/지갑을 선택한 실제 반영 거래는 계좌 계산 잔액에 반영된다.

## 일반 예정 거래

- [ ] `예정` 상태로 미래 날짜 거래를 저장할 수 있다.
- [ ] 예정 거래는 `scheduled` 상태로 저장된다.
- [ ] 예정 거래는 홈 거래 목록과 전체보기 목록에 `예정`으로 표시된다.
- [ ] 예정 거래는 실제 월 수입/지출 합계와 월 정산에 포함되지 않는다.
- [ ] 예정 거래는 계좌/지갑 계산 잔액에 포함되지 않는다.
- [ ] 예정 거래는 예정 수입/예정 지출/월말 예상 잔액에는 반영된다.
- [ ] 예정 거래 상세 화면에서 상태가 구분되어 보인다.
- [ ] 예정 거래를 수정해도 기존 정책이 유지된다.
- [ ] 예정 거래 삭제 후 홈, 전체보기, 계좌 예정 금액이 갱신된다.

## 예정 거래 날짜 검증

- [ ] `예정 + 과거 날짜` 저장 시 저장되지 않고 에러 메시지가 표시된다.
- [ ] `예정 + 오늘 날짜` 저장 시 저장되지 않고 에러 메시지가 표시된다.
- [ ] `예정 + 내일 이후 날짜` 저장 시 정상 저장된다.
- [ ] 수정 화면에서도 `scheduled + 오늘/과거 날짜` 저장이 차단된다.
- [ ] `posted` 거래는 과거/오늘/미래 날짜 모두 저장 가능하다.
- [ ] 잘못된 날짜 형식은 기존 날짜 형식 에러로 처리된다.

## 반복 거래 등록/수정/비활성화/삭제

- [ ] 설정 화면에서 `반복 거래 관리`로 이동할 수 있다.
- [ ] 반복 거래 추가 화면을 열 수 있다.
- [ ] 반복 거래 새 등록 기본 시작일이 현재 기준 날짜 다음 날로 설정된다.
- [ ] 반복 거래 시작일이 오늘/과거이면 저장이 차단된다.
- [ ] 반복 거래 시작일이 미래이면 저장된다.
- [ ] 무료 상태에서는 활성 반복 거래 제한이 적용된다.
- [ ] 프리미엄 상태에서는 반복 거래 제한이 완화된다.
- [ ] 반복 거래 수정 시 기존 값이 폼에 표시된다.
- [ ] 기존 과거 시작일 원본은 시작일을 바꾸지 않는 수정은 가능하다.
- [ ] 시작일을 오늘/과거로 변경하려 하면 저장이 차단된다.
- [ ] 반복 거래 비활성화 후 더 이상 새 occurrence가 생성되지 않는다.
- [ ] 비활성 반복 거래를 다시 사용할 수 있다.
- [ ] 반복 거래 원본 삭제 후 기존 생성 거래는 유지된다.
- [ ] 삭제한 occurrence는 재생성 방지 정책이 유지된다.

## scheduled -> posted 전환

- [ ] 기준 날짜보다 미래인 scheduled 거래는 scheduled 상태를 유지한다.
- [ ] 기준 날짜가 scheduled 거래 날짜에 도달하면 동기화 후 posted로 전환된다.
- [ ] 기준 날짜가 scheduled 거래 날짜를 지난 경우 동기화 후 posted로 전환된다.
- [ ] 전환된 거래는 홈 월 정산과 계좌 계산 잔액에 반영된다.
- [ ] 반복 거래 occurrence 생성 시 `occurrence date > today`는 scheduled가 된다.
- [ ] 반복 거래 occurrence 생성 시 `occurrence date <= today`는 posted가 된다.
- [ ] 기존 scheduled -> posted 전환 후 중복 거래가 생성되지 않는다.

## Debug 테스트 날짜

- [ ] Debug 빌드 설정 화면 하단에 `개발 도구` 섹션이 보인다.
- [ ] 현재 기준 날짜가 표시된다.
- [ ] 테스트 날짜 선택 DatePicker가 열린다.
- [ ] 테스트 날짜 저장 후 기준 날짜 표시가 바뀐다.
- [ ] 테스트 날짜 기준으로 예정 거래 날짜 검증 결과가 바뀐다.
- [ ] 테스트 날짜 기준으로 반복 거래 시작일 검증 결과가 바뀐다.
- [ ] `예정/반복 거래 동기화 실행` 버튼으로 scheduled -> posted 전환을 실행할 수 있다.
- [ ] 실제 날짜로 되돌리기 버튼이 동작한다.
- [ ] Release 빌드에서는 개발 도구 섹션이 보이지 않는다.
- [ ] Release 빌드에서는 저장된 테스트 날짜가 있어도 실제 날짜 기준으로 동작한다.

## 계좌/지갑 잔액 계산

- [ ] 설정 화면과 홈 계좌/지갑 카드에서 계좌/지갑 관리 화면으로 이동할 수 있다.
- [ ] 계좌/지갑을 추가할 수 있다.
- [ ] 계좌/지갑을 수정할 수 있다.
- [ ] 계좌/지갑을 비활성화할 수 있다.
- [ ] 비활성 계좌/지갑을 다시 사용할 수 있다.
- [ ] 계좌/지갑을 삭제할 수 있다.
- [ ] 거래가 연결된 계좌 삭제 시 거래 상세에는 snapshot 정보가 유지된다.
- [ ] 무료 상태에서는 계좌/지갑 등록 제한이 적용된다.
- [ ] 프리미엄 상태에서는 계좌/지갑 제한이 완화된다.
- [ ] 계좌 계산 잔액은 기준 잔액 + posted 수입 - posted 지출로 계산된다.
- [ ] scheduled 거래는 계좌 계산 잔액에 포함되지 않는다.
- [ ] scheduled 거래는 계좌의 예정 수입/예정 지출/월말 예상 잔액에 반영된다.
- [ ] 홈 계좌/지갑 카드에는 활성 계좌만 표시된다.
- [ ] 계좌가 많으면 홈에는 일부만 표시되고 전체 계좌 보기로 이동할 수 있다.
- [ ] 비활성 계좌와 삭제된 계좌는 홈 요약에 표시되지 않는다.

## 홈 월 정산

- [ ] 홈 월 요약 카드 제목이 `이번 달 정산`으로 표시된다.
- [ ] 월 정산 금액은 posted 수입 합계 - posted 지출 합계로 계산된다.
- [ ] scheduled 거래는 월 정산에 포함되지 않는다.
- [ ] 수입이 지출보다 많으면 `+금액`과 남았다는 안내가 표시된다.
- [ ] 지출이 수입보다 많으면 `-금액`과 초과 지출 안내가 표시된다.
- [ ] 수입과 지출이 같으면 0원 안내가 표시된다.
- [ ] 이전/다음 월 이동이 동작한다.
- [ ] 월 텍스트 클릭으로 년/월 선택 다이얼로그가 열린다.
- [ ] 1900년~2100년 범위에서 월을 선택할 수 있다.
- [ ] 선택 월 기준 거래 목록과 월 정산이 갱신된다.

## 전체보기 월/연도 정산

- [ ] 전체보기 화면에 모든 거래가 월별로 그룹화된다.
- [ ] 최신 연도와 최신 월이 위에 표시된다.
- [ ] 월 내부 거래는 최신 날짜순으로 표시된다.
- [ ] 같은 날짜 거래는 최신 작성순으로 표시된다.
- [ ] 연도 요약에는 총 수입, 총 지출, 정산 금액이 표시된다.
- [ ] 월 요약에는 총 수입, 총 지출, 월 정산이 표시된다.
- [ ] 전체보기에서 `월 잔액`, `연도 잔액` 같은 오용 문구가 보이지 않는다.
- [ ] 전체보기 거래 항목 클릭 시 상세 화면으로 이동한다.
- [ ] 거래가 없으면 빈 상태 문구가 표시된다.

## 영수증 첨부/확대

- [ ] 지출 작성 화면에서 image 파일을 선택할 수 있다.
- [ ] 선택한 영수증 사진 미리보기가 표시된다.
- [ ] 8MB 초과 이미지는 저장되지 않고 에러 메시지가 표시된다.
- [ ] 저장 후 상세 화면에 영수증 사진이 표시된다.
- [ ] 앱 재실행 후에도 영수증 사진이 표시된다.
- [ ] 수정 화면에서 기존 영수증 사진이 표시된다.
- [ ] 수정 화면에서 영수증 사진을 변경할 수 있다.
- [ ] 수정 화면에서 영수증 사진을 삭제할 수 있다.
- [ ] 상세 화면에서 영수증 사진을 누르면 확대 보기 화면이 열린다.
- [ ] 확대 보기에서 pinch zoom이 동작한다.
- [ ] 확대 상태에서 드래그 이동이 동작한다.
- [ ] 닫기 버튼과 뒤로가기로 확대 보기를 닫을 수 있다.
- [ ] 이미지를 한 번 터치해도 바로 닫히지 않는다.

- [ ] 무료/프리미엄 상태와 무관하게 JSON 내보내기가 가능하다.
- [ ] 백업 파일명이 `hannun-ledger-backup-YYYY-MM-DD.json` 형식이다.
- [ ] v3 백업에 거래, 계좌/지갑, 반복 거래, skipped occurrence 정보가 포함된다.
- [ ] 영수증 이미지 파일은 JSON 백업에 포함되지 않는다.
- [ ] 복원 시 `receiptImagePath`는 null 처리되거나 무시된다.
- [ ] v1 거래-only 백업을 가져올 수 있다.
- [ ] v2 거래+계좌 백업을 가져올 수 있다.
- [ ] v3 백업을 가져올 수 있다.
- [ ] 병합 가져오기는 같은 id 데이터를 중복 추가하지 않는다.
- [ ] 교체 가져오기는 확인 다이얼로그를 거친다.
- [ ] 교체 후 기존 거래/계좌/반복 거래가 백업 데이터로 대체된다.
- [ ] 잘못된 JSON, 빈 파일, 다른 앱 JSON은 에러 메시지를 표시하고 기존 데이터를 유지한다.
- [ ] 전체 데이터 삭제는 확인 다이얼로그를 거친다.
- [ ] 전체 데이터 삭제 후 홈/전체보기/계좌/반복 거래 화면이 빈 상태로 갱신된다.

## 백업/복원 트랜잭션 회귀 테스트

- [ ] 복원 전 기존 거래, 계좌/지갑, 반복 거래, 영수증 사진이 있는 상태를 준비한다.
- [ ] 병합 가져오기 성공 후 새 거래/계좌/반복 거래만 추가되고 기존 데이터는 유지된다.
- [ ] 병합 가져오기 성공 후 같은 id 거래/계좌/반복 거래가 중복 추가되지 않는다.
- [ ] 병합 가져오기 성공 후 skipped occurrence가 유지되어 삭제한 반복 occurrence가 재생성되지 않는다.
- [ ] 병합 가져오기 성공 후 scheduled -> posted 전환과 반복 occurrence 생성이 정상 동작한다.
- [ ] 교체 가져오기 성공 후 기존 거래/계좌/반복 거래는 백업 데이터로 완전히 대체된다.
- [ ] 교체 가져오기 성공 후 홈 월 정산, 전체보기, 계좌 계산 잔액, 반복 거래 목록이 복원 데이터 기준으로 갱신된다.
- [ ] 교체 가져오기 성공 후 백업에 포함되지 않은 영수증 사진 경로는 복원 거래에 남지 않는다.
- [ ] 교체 가져오기 성공 후 기존 거래에 연결된 영수증 이미지 파일이 가능한 범위에서 정리된다.
- [ ] 잘못된 JSON, 빈 파일, 다른 앱 JSON 가져오기 실패 후 기존 거래/계좌/반복 거래가 유지된다.
- [ ] 가져오기 실패 후 홈 월 정산, 계좌 계산 잔액, 반복 거래 목록이 실패 전과 동일하다.
- [ ] 전체 데이터 삭제 성공 후 거래, 계좌/지갑, 반복 거래, skipped occurrence가 모두 삭제된다.
- [ ] 전체 데이터 삭제 성공 후 기존 영수증 이미지 파일이 가능한 범위에서 정리된다.
- [ ] 전체 데이터 삭제 후 앱을 재실행해도 빈 상태가 유지된다.
- [ ] 복원/삭제 도중 오류 메시지가 표시되어도 앱이 크래시 나지 않는다.
- [ ] 영수증 이미지 파일 삭제는 DB transaction 밖에서 처리되므로, 파일 정리 실패가 DB 복원 성공을 되돌리지 않는다는 한계를 확인한다.

## 월별/연도별 PDF

- [ ] 홈 또는 전체보기에서 월별 PDF 생성 흐름을 실행할 수 있다.
- [ ] 전체보기에서 연도별 PDF 생성 흐름을 실행할 수 있다.
- [ ] 무료 사용자는 PDF 3회 체험 제한이 적용된다.
- [ ] 프리미엄 사용자는 PDF를 제한 없이 생성할 수 있다.
- [ ] PDF 생성 성공 후 공유 Intent가 열린다.
- [ ] PDF에는 영수증 이미지가 포함되지 않는다.
- [ ] 월별 PDF에 월 제목, 총 수입, 총 지출, 월 정산, 거래 목록, 생성일이 표시된다.
- [ ] 연도별 PDF에 연도 제목, 총 수입, 총 지출, 연도 정산, 월별 섹션이 표시된다.
- [ ] PDF에서 `잔액` 오용 문구가 보이지 않는다.
- [ ] 거래가 많은 월/연도는 여러 페이지로 나뉜다.
- [ ] 긴 제목/카테고리는 PDF 레이아웃을 깨지 않는다.

## 광고/프리미엄 상태

- [ ] 무료 상태에서 AdMob 테스트 상단 배너가 표시된다.
- [ ] 프리미엄 상태에서 광고 배너가 숨겨진다.
- [ ] 설정 화면 프리미엄 섹션에 현재 상태가 표시된다.
- [ ] Debug 빌드에서 프리미엄 상태 전환 버튼이 보인다.
- [ ] Debug 빌드에서 개발용 프리미엄 전환 버튼으로 프리미엄 상태를 켜고 끌 수 있다.
- [ ] Release 빌드에서는 개발용 프리미엄 전환 버튼이 보이지 않는다.
- [ ] Release 빌드에서는 Debug 빌드에서 저장한 프리미엄 상태가 남아 있어도 무료 상태로 판정된다.
- [ ] Release 빌드에서는 Billing 연결 전 프리미엄 상태가 자동 활성화되지 않는다.
- [ ] 프리미엄 안내 문구에 광고 제거, PDF 무제한, 계좌/반복 거래 확장 내용이 표시된다.
- [ ] 실제 Google Play Billing 결제 기능은 연결되어 있지 않다.

## 설정 화면

- [ ] 홈 우측 상단 설정 버튼으로 설정 화면에 진입한다.
- [ ] 데이터 관리 섹션이 표시된다.
- [ ] 계좌/지갑 관리 진입 버튼이 표시된다.
- [ ] 반복 거래 관리 진입 버튼이 표시된다.
- [ ] 영수증 섹션에 현재 첨부된 사진 수가 표시된다.
- [ ] 앱 설정 섹션에 라이트 모드 고정 안내가 표시된다.
- [ ] 프리미엄 섹션이 표시된다.
- [ ] 앱 정보 섹션에 앱 이름, versionName, versionCode, package가 표시된다.
- [ ] 뒤로 버튼으로 이전 화면에 복귀한다.

## 상세/수정/삭제

- [ ] 홈 거래 카드 클릭 시 상세 화면으로 이동한다.
- [ ] 전체보기 거래 항목 클릭 시 상세 화면으로 이동한다.
- [ ] 상세 화면에 종류, 상태, 제목, 금액, 날짜, 카테고리, 메모, 계좌가 표시된다.
- [ ] 긴 제목과 긴 메모가 화면 밖으로 넘치지 않는다.
- [ ] 수정 화면에 기존 거래값이 채워진다.
- [ ] 수정 저장 후 홈, 전체보기, 계좌 계산값이 갱신된다.
- [ ] 삭제 버튼 클릭 시 확인 다이얼로그가 표시된다.
- [ ] 삭제 확인 후 거래가 즉시 사라진다.
- [ ] 반복 거래 occurrence 삭제 시 재생성 방지 정책이 유지된다.

## 앱 재실행 후 데이터 유지

- [ ] 앱을 완전히 종료하고 다시 실행한다.
- [ ] Room 거래 데이터가 유지된다.
- [ ] 계좌/지갑 데이터가 유지된다.
- [ ] 반복 거래 원본 데이터가 유지된다.
- [ ] scheduled/posted 상태가 유지된다.
- [ ] 홈 월 정산이 재실행 후 동일하다.
- [ ] 계좌 계산 잔액이 재실행 후 동일하다.
- [ ] 영수증 사진이 상세 화면에서 다시 표시된다.
- [ ] JSON 백업/복원 이후 재실행해도 데이터가 유지된다.

## 에뮬레이터/실기기 연결 점검

- [ ] `adb devices`에서 대상 기기가 `device` 상태로 표시된다.
- [ ] `offline`, `unauthorized` 상태가 아니다.
- [ ] Photo Picker가 열린다.
- [ ] JSON 저장 위치 선택 화면이 열린다.
- [ ] JSON 파일 선택 화면이 열린다.
- [ ] PDF 공유 화면이 열린다.
- [ ] 앱 삭제 후 재설치 시 로컬 데이터 초기화 정책이 예상과 일치한다.
