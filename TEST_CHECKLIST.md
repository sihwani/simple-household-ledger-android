# 한눈 가계부 Android MVP 테스트 체크리스트

이 문서는 Android Studio, 에뮬레이터, 실제 Android 기기에서 MVP 기능을 확인하기 위한 실행 체크리스트입니다.

## 현재 구현된 기능

- 홈 화면: 선택 월 기준 잔액, 수입 합계, 지출 합계 표시
- 월 이동: 이전 달/다음 달 이동
- 수입 작성: 금액, 수입명, 카테고리, 날짜, 메모 저장
- 지출 작성: 금액, 사용처, 카테고리, 날짜, 메모, 영수증 사진 저장
- 상세 화면: 거래 상세 정보 표시
- 수정: 기존 거래값을 불러와 수정 후 저장
- 삭제: 확인 다이얼로그 후 단일 거래 삭제
- 전체보기: 모든 거래를 월별 그룹으로 표시
- 월별 그룹 요약: 월 총 수입, 월 총 지출, 월 잔액 표시
- 영수증: 지출 작성/수정에서 사진 첨부, 상세 표시, 확대 보기
- JSON 백업: 거래 데이터 내보내기
- JSON 복원: 병합/교체 방식 가져오기
- 전체 데이터 삭제: 모든 거래 및 연결된 내부 영수증 파일 삭제

## 첫 실행 테스트

- [ ] 앱이 크래시 없이 실행된다.
- [ ] 앱 이름이 `한눈 가계부`로 표시된다.
- [ ] 홈 화면에 현재 월이 표시된다.
- [ ] 거래가 없을 때 수입/지출 빈 상태가 자연스럽게 보인다.
- [ ] `전체 내역 보기` 버튼으로 전체보기 화면에 진입할 수 있다.
- [ ] 전체보기에서 거래가 없을 때 빈 상태 문구가 보인다.
- [ ] 데이터 관리 섹션이 전체보기 하단에 보인다.

## 수입 작성 테스트

- [ ] 홈에서 `수입` 버튼을 눌러 수입 작성 화면으로 이동한다.
- [ ] 기본 날짜가 실제 오늘 날짜로 표시된다.
- [ ] 금액이 비어 있거나 0원이면 에러 메시지가 표시된다.
- [ ] 수입명이 비어 있으면 에러 메시지가 표시된다.
- [ ] 잘못된 날짜 형식이면 에러 메시지가 표시된다.
- [ ] 정상 입력 후 저장하면 홈으로 돌아온다.
- [ ] 홈 월별 수입 합계와 잔액에 반영된다.
- [ ] 전체보기 월별 그룹에도 표시된다.
- [ ] 수입 작성 화면에는 영수증 사진 섹션이 보이지 않는다.

## 지출 작성 및 영수증 테스트

- [ ] 홈에서 `지출` 버튼을 눌러 지출 작성 화면으로 이동한다.
- [ ] 기본 날짜가 실제 오늘 날짜로 표시된다.
- [ ] 영수증 사진 섹션이 보인다.
- [ ] `사진 선택` 버튼으로 Photo Picker가 열린다.
- [ ] 이미지를 선택하면 작성 화면에 미리보기가 표시된다.
- [ ] 저장 후 홈 지출 목록, 지출 합계, 잔액에 반영된다.
- [ ] 상세 화면에서 영수증 사진이 표시된다.
- [ ] 영수증 사진을 누르면 어두운 배경의 확대 보기 화면이 열린다.
- [ ] 확대 보기에서 닫기 버튼 또는 뒤로가기로 상세 화면에 복귀한다.
- [ ] 8MB 초과 이미지를 저장하려 하면 에러 메시지가 표시된다.

## 상세/수정/삭제 테스트

- [ ] 홈 거래 카드를 누르면 상세 화면으로 이동한다.
- [ ] 전체보기 거래 항목을 누르면 상세 화면으로 이동한다.
- [ ] 상세 화면에 종류, 제목, 금액, 날짜, 카테고리, 메모가 표시된다.
- [ ] 긴 제목과 긴 메모가 화면 밖으로 넘치지 않는다.
- [ ] `수정` 버튼으로 수정 화면에 진입한다.
- [ ] 기존 거래값이 폼에 미리 채워진다.
- [ ] 지출 수정 화면에는 기존 영수증 이미지가 보인다.
- [ ] 지출 수정에서 사진 변경 후 저장하면 상세 화면에 변경된 이미지가 보인다.
- [ ] 지출 수정에서 사진 삭제 후 저장하면 상세 화면에 영수증 영역이 사라진다.
- [ ] 수정 저장 후 홈/전체보기 값이 최신으로 반영된다.
- [ ] `삭제` 버튼을 누르면 확인 다이얼로그가 표시된다.
- [ ] 삭제 확인 후 홈으로 돌아오고 거래가 목록에서 사라진다.

## 전체보기 월별 그룹 테스트

- [ ] 여러 월의 거래를 작성한다.
- [ ] 전체보기에서 최신 월이 위에 표시된다.
- [ ] 월 내부 거래가 날짜 최신순으로 표시된다.
- [ ] 같은 날짜 거래는 나중에 작성한 거래가 위에 표시된다.
- [ ] 각 월의 총 수입, 총 지출, 월 잔액이 정확하다.
- [ ] 큰 금액도 화면 밖으로 넘치지 않는다.

## JSON 백업/복원 테스트

- [ ] 전체보기 하단에서 `데이터 내보내기`를 누른다.
- [ ] 저장 위치를 선택하면 `hannun-ledger-backup-YYYY-MM-DD.json` 파일이 생성된다.
- [ ] 백업 JSON에 `app`, `version`, `exportedAt`, `transactions`가 포함된다.
- [ ] 백업 거래의 `receiptImagePath`는 `null`이다.
- [ ] `전체 데이터 삭제`를 누르면 확인 다이얼로그가 표시된다.
- [ ] 삭제 확인 후 홈/전체보기가 빈 상태로 갱신된다.
- [ ] `데이터 가져오기`로 방금 내보낸 JSON을 선택한다.
- [ ] 병합을 선택하면 기존 ID와 중복되지 않는 거래만 추가된다.
- [ ] 교체를 선택하면 확인 다이얼로그 후 백업 데이터로 전체 교체된다.
- [ ] 잘못된 JSON, 빈 파일, 다른 앱 JSON은 에러 메시지를 표시하고 기존 데이터를 유지한다.
- [ ] 가져오기 후 영수증 사진은 복원되지 않는다는 점이 UI 안내와 일치한다.

## 앱 재실행 테스트

- [ ] 앱을 완전히 종료하고 다시 실행한다.
- [ ] Room에 저장된 거래 데이터가 유지된다.
- [ ] 홈 월별 요약이 재실행 후에도 동일하다.
- [ ] 전체보기 월별 그룹이 재실행 후에도 동일하다.
- [ ] 내부 저장소에 저장된 영수증 사진이 상세 화면에서 다시 표시된다.

## 에뮬레이터 실행 점검

- [ ] Android Studio Device Manager에서 에뮬레이터가 정상 생성되어 있다.
- [ ] 에뮬레이터가 부팅 완료 상태다.
- [ ] `adb devices`에서 `emulator-xxxx device`로 표시된다.
- [ ] `offline`, `unauthorized`가 아닌 `device` 상태다.
- [ ] 앱 설치 후 Photo Picker, 파일 내보내기, 파일 가져오기 화면이 열린다.
- [ ] 에뮬레이터 저장소에 백업 JSON을 저장하고 다시 가져올 수 있다.

## 실제 기기 USB 디버깅 테스트

- [ ] 스마트폰 개발자 옵션이 활성화되어 있다.
- [ ] USB 디버깅이 켜져 있다.
- [ ] PC 연결 후 스마트폰에서 RSA 디버깅 허용 팝업을 승인한다.
- [ ] `adb devices`에서 기기 시리얼이 `device` 상태로 표시된다.
- [ ] Android Studio 실행 대상에서 실제 기기를 선택할 수 있다.
- [ ] 사진 선택, 내부 저장소 저장, JSON 파일 선택/저장이 실제 기기에서도 동작한다.

## ADB 연결 문제 우회 방법

- [ ] USB 케이블을 데이터 전송 지원 케이블로 교체한다.
- [ ] 스마트폰 USB 연결 모드를 `파일 전송`으로 바꾼다.
- [ ] `adb kill-server` 후 `adb start-server`를 실행한다.
- [ ] Android Studio Device Manager에서 에뮬레이터를 Cold Boot 한다.
- [ ] Windows 장치 관리자에서 Android USB Driver 설치 상태를 확인한다.
- [ ] 에뮬레이터가 계속 안 잡히면 Android Studio의 `Build APK(s)`로 APK를 생성한 뒤 실제 기기에 직접 설치해 기능을 확인한다.
